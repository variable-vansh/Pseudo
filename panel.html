            How metrics work
            
          

          

            
Input tokens

            
Exact prompt token count returned by the model's API after generation. Includes your pseudocode and the locked system prompt.


            
Output tokens

            
Exact completion token count returned by the API — the tokens used to generate the code you see.


            
Thinking tokens

            
Shown only for Gemini thinking models. These are internal reasoning tokens used before the response — not visible in the generated code.


            
Efficiency Score

            
Starts at 100. Loses 15 pts per iteration beyond the first. Loses 0.05 pts per token above 120 average input. Gains 20 pts for a single-attempt solve. Clamped 0–100.


            
Session locked

            
Freezes the session record: no more iterations are added. Happens automatically on an Accepted verdict, or manually via the Lock button. The score is final after locking.
