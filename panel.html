            How metrics work
            
          

          

            
Input tokens

            
The number of tokens in everything sent to the model on each call: your pseudocode plus the system prompt Pseudo adds automatically.


            
Output tokens

            
Tokens the model generated one at a time to produce the code you see. Reported by the API after every call.


            
Thinking tokens

            
Only shown for models that reason before responding. These internal chain-of-thought tokens are billed but never appear in the output.


            
Resource Units

            
A compute-weighted score based on model size and token usage. It tracks real inference cost better than raw token count: larger models do more work per token, and output is weighted 3× over input because it is generated autoregressively. See the ? in the session summary or next to + Budget for details.


            
Resource Unit budget

            
An optional cap set before your first generate. Progress is measured in Resource Units, not tokens, so the limit stays meaningful even if you switch models.


            
Session ended

            
Locks the session so no more calls are recorded. Happens when you click End session, or automatically on an Accepted verdict.
